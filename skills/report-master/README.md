## 🎯 这是什么？

**报告老油条** 是一个专为当代打工人设计的 AI 技能，帮你自动生成各种职场报告：

- ✅ 周报/月报/季报/年报
- ✅ 培训总结/学习报告
- ✅ 项目复盘/工作汇报
- ✅ 任何需要"交差"的文档

**核心理念：** 不吹牛、能交差、不糊弄，是当代打工人的标杆。

---

## ✨ 核心特性

### 🎭 老油条心法

| 原则 | 说明 |
|------|------|
| 不吹牛 | 数据可以模糊，但绝不编造 |
| 能交差 | 该有的部分都有，该有的流程都走 |
| 不糊弄 | 看起来专业，术语用得恰到好处 |
| 有退路 | "持续优化""稳步推进""进一步完善" |
| 甩锅于无形 | "受限于外部因素""待跨部门协同后推进" |

### 📐 格式为王

- **严格遵循模板**：字体、字号、框线、对齐、单元格合并、缩进、行距——一个都不能错
- **格式跟随**：给什么格式，输出什么格式（DOCX/XLSX/飞书文档/飞书表格）
- **逐像素还原**：领导第一眼看的永远是格式，格式对了就成功 80%

### 🗣️ 经典话术库

```
开头套路：
"在本周期内，团队围绕核心目标，稳步推进各项工作..."
"根据公司战略部署，结合业务实际情况..."
"在领导的正确指导和团队共同努力下..."

中间套路：
"完成阶段性目标，部分指标呈现积极向好态势"
"针对存在的问题，已制定专项改进计划"
"通过优化流程/细化分工/加强协同，工作效率有所提升"

结尾套路：
"下阶段将继续保持当前工作节奏，重点关注..."
"针对上述不足，将制定切实可行的改进措施，确保各项工作有序推进"
"恳请领导批评指正"
```

## 🚀 快速开始

### 前置要求

- Python 3.8+
- `python-docx` (生成 DOCX)
- `openpyxl` (生成 XLSX)

### 安装依赖

```bash
pip install python-docx openpyxl
```

### 使用示例

#### 1. 生成标准周报

```bash
python scripts/generate_report.py --title "工作周报" --week "2026 年第 15 周" --output report.docx
```

#### 2. 生成学习报告（带模板）

```bash
python scripts/generate_study_report.py template.xlsx output.xlsx
```

#### 3. 作为 OpenClaw Skill 使用

将本目录放入 OpenClaw 的 skills 文件夹，然后直接对话：

```
用户：帮我写个周报
报告老油条：好的，请问：
1. 这周主要做了什么工作？
2. 有没有模板格式要求？
3. 需要突出哪些成果？
```

## 💡 使用场景

### 场景 1：被派去培训，啥都没学到

```
输入：被公司派去参加了 AI 分享会议，但我觉得啥都没学到，
      现在公司让我写报告

输出：一份格式完美、内容"充实"的学习报告
      - 课前思考：5 个看起来很深入的问题
      - 课中记录：4 大模块，正确的废话
      - 课后转化：PDCA 闭环，行动计划都是"调研"、"讨论"、"配合"
      - 评价栏：全部"待实施后填写"
```

### 场景 2：周报写不出来

```
输入：这周就摸了 5 天鱼，怎么写周报？

输出：
"本周工作围绕季度目标稳步推进，各项任务按计划有序开展。
重点推进了 XX 项目的阶段性交付，目前已完成约 70% 的进度。
针对前期识别的风险点，已协调相关资源进行专项处理，整体风险可控。"
```

### 场景 3：有模板要填

```
输入：[上传 Excel 模板] 帮我填一下这个

输出：严格按模板格式还原，内容老油条化
      - 合并单元格位置一致
      - 列宽、行高匹配
      - 边框、字体完整
```

## ⚠️ 免责声明

本工具生成的内容仅供娱乐和参考，请根据实际情况调整使用。

**不建议使用场景：**
- 需要真实数据的正式汇报
- 涉及法律/财务/医疗等专业领域
- 老板真的会仔细看的内容

**建议使用场景：**
- 形式主义的流程文档
- 交差为主的例行报告
- 需要"看起来很专业"的场合
