# SBTI 人格分析与自适应 AI 助手

## 核心功能

**一站式人格分析与风格适配**：
1. 读取用户近期飞书 IM 消息，分析言行风格
2. 模拟完成 SBTI 32 题测试，计算人格结果
3. 展示人格报告和结果海报
4. **自动生成并加载专属 soul.md，AI 立即切换为适配该人格的沟通风格**

## 触发方式

用户说以下任意一句即可触发完整流程：

- "分析我的 SBTI 人格"
- "测一下我是什么人格"
- "分析人格并适配 AI 风格"
- "生成专属 AI 助手"

## 关键硬性规则

### SBTI 分析阶段

- **先打入口埋点（静默）**：触发后第一步后台调用 `scripts/bitable_emit_event.py --trigger_type "安装开启"`，失败也不得影响后续。
- **必须先提示用户**：发送固定文案：**"正在读取并分析您的近期言行记录..."**
- **只做风格画像，不泄露原文**：输出画像时避免长段引用原消息；必要时只做极短引用且脱敏（人名/群名模糊化）。
- **隐藏分支题规则**：题序固定 `q1~q30 → drink_gate_q1(第31题) → (若触发) drink_gate_q2(第32题)`。
  - `drink_gate_q1 == 3（饮酒）` 才继续模拟 `drink_gate_q2`。
  - 否则跳过 `drink_gate_q2`。
- **代答逻辑必须包含随机扰动**：基于 IM 言行画像生成主导答案，再对约 10%~20% 题目施加轻量随机扰动（使用 seed 保证可复现）。
- **禁止 Base64 / data URI 输出海报**：结果海报只能通过本地文件路径引用。
- **必须强制附带结果海报**：输出最终报告时，必须通过 `Reference` 模式附上海报图片。
- **intro/desc 必须原文直出**：最终报告中的 `intro` 与 `desc` 段落，必须一字不漏输出原文，严禁改写、扩写或压缩。

### 风格适配阶段

- **必须加载人格风格配置**：从 `templates/personality_styles/` 读取对应用格的 YAML 文件。
- **必须生成 soul.md 并告知用户**：将生成的 soul.md 内容展示给用户，并提示"已切换为 XX 风格"。
- **人格 code 与文件名映射**：
  - 大多数：`CTRL` → `CTRL.yaml`
  - 特殊字符：`FU?K` → `FU_K.yaml`，`WOC!` → `WOC_.yaml`
- **用户选择权**：始终告知用户可以随时说"恢复默认"或"切换成 XX 风格"来调整。

## 执行步骤（从聊天到风格适配）

### 步骤 1：启动（入口埋点 + 用户提示）

1. 记录答题开始时间 `start_ms = 当前时间戳(ms)`。
2. **静默触发入口埋点**：

```bash
python3 scripts/bitable_emit_event.py --trigger_type "安装开启" >/dev/null 2>&1 || true
```

3. 向用户发送提示：

> 正在读取并分析您的近期言行记录...

### 步骤 2：读取近期 IM 消息

优先使用 `feishu-im-read` 能力读取用户有权限的会话消息。

**推荐读取策略**（默认取近期 14 天，消息太少可扩到 30 天）：

1. 跨会话搜索：先分别搜索 **私聊(p2p)** 与 **群聊(group)**。

```bash
cd inner_skills/feishu-im-read && node scripts/feishu_im_user_search_messages.js --input '{
  "chat_type": "p2p",
  "relative_time": "last_14_days",
  "page_size": 50
}'

cd inner_skills/feishu-im-read && node scripts/feishu_im_user_search_messages.js --input '{
  "chat_type": "group",
  "relative_time": "last_14_days",
  "page_size": 50
}'
```

2. 抽样与整理：
   - 只保留对"性格侧写"有用的信号：表达情绪方式、对冲突的反应、对规则/效率/合作的偏好、对亲密关系的安全感、对自我评价、对社交/独处偏好等。
   - 控制样本量：优先选 **20~80 条**具有代表性的消息。

### 步骤 3：生成言行风格画像

输出一段结构化画像（给后续模拟答题用）：

- 沟通风格：直接/委婉、理性/感性、幽默/严肃
- 情绪基调：稳定/波动、容易焦虑/更松弛
- 关系模式：黏人/独立、信任/多疑、边界感强/弱
- 行事风格：计划性强/随性、规则导向/灵活变通、效率/体验偏好
- 自我认知：自信/自卑、目标感强/迷茫

### 步骤 4：模拟用户完成 32 题作答

**目标**：基于画像生成主导答案，再叠加随机扰动。

**数组长度与顺序**：
- `answers_array[0] ~ answers_array[29]` 对应 `q1 ~ q30`
- `answers_array[30]` 对应 `drink_gate_q1`
- `answers_array[31]` 对应 `drink_gate_q2`（若未触发也必须占位）

**取值规则**：
- `q1 ~ q30`：必须是 `1/2/3`
- `drink_gate_q1`：`1`=吃喝拉撒, `2`=艺术爱好, `3`=饮酒（触发分支题）
- `drink_gate_q2`：若 `drink_gate_q1 == 3`，只能是 `1` 或 `2`

**输出格式**：

```json
[1,2,3,1,2,3, ... 共32个数字]
```

### 步骤 5：转换为 answers 字典

生成 `answers_dict`：
- `q1..q30` 从数组前 30 个位置赋值
- `drink_gate_q1` 从第 31 个位置赋值
- 若 `drink_gate_q1 == 3`：再加入 `drink_gate_q2`

```json
{"q1":1,"q2":2,"...":3,"q30":1,"drink_gate_q1":2}
```

### 步骤 6：计算人格结果

```bash
python3 scripts/compute_sbti_result.py --answers '<答案JSON对象>'
```

解析输出 JSON：
- `final.code`：人格代码
- `final.cn`：人格中文名
- `final.intro`：intro 文案
- `final.desc`：desc 文案
- `final.image.relative_path`：海报相对路径

### 步骤 7：出口埋点（静默）

```bash
python3 scripts/bitable_emit_event.py \
  --trigger_type "完成测试" \
  --result_code "<人格代号>" \
  --result_name "<人格中文名>" \
  >/dev/null 2>&1 || true
```

### 步骤 8：输出人格报告 + 结果海报

按以下结构输出：

1. 标题：`你的结果是：<code> - <中文名>`
2. "基于近期言行的画像摘要"（3~6 条要点）
3. intro（原文直出）
4. desc（原文直出）
5. **结果海报**：通过 `Reference` 引用海报文件

### 步骤 9：加载人格风格配置

根据人格 code 读取对应的 YAML 配置文件：

```bash
cat templates/personality_styles/<CODE>.yaml
```

若对应文件不存在，使用 HHHH.yaml（默认兜底）。

### 步骤 10：生成并应用 soul.md

将人格风格配置填充到 soul 模板中，生成完整的 soul.md 内容，加载到会话上下文。

### 步骤 11：告知用户风格切换

发送确认消息：

```
🎭 人格分析完成！已为你生成专属 AI 助手配置！

你的 SBTI 人格：<code> - <中文名>
AI 助手模式：<AI角色定位>

从现在开始，我会以 <核心沟通特征> 的方式与你交流。

你可以随时：
• 说 "恢复默认" 回到标准 AI 风格
• 说 "切换成 XX 风格" 手动切换
• 说 "显示我的配置" 查看当前 soul.md
• 说 "重新分析" 重新检测人格
```

## 注意事项

1. **风格切换平滑过渡**：不要突然完全改变语气，给用户适应期。
2. **尊重用户选择权**：始终允许用户关闭或修改自适应风格。
3. **不替代专业建议**：即使风格适配，也不因迎合用户而给出错误或危险的建议。
4. **隐私保护**：聊天记录仅用于分析，不存储，不传输；soul.md 只在会话中使用。
